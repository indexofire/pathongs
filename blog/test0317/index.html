    真核生物基因组的基因预测和注释¶

    1 用maker进行真核基因组的基因预测¶

    maker结合了重复序列屏蔽，augustus，genemark，snap从头预测，同源注释以及RNA-seq的方法，其本身并不属于一种预测基因的方法，其为一套整合各个方法的流程。

    1.1 获取maker的配置文件¶

    maker -CTL

    配置文件¶

    • maker_bopts.ctl 可不用修改

    • maker_exe.ctl 执行路径

    • maker_opts.ctl 重点修改对象

    1.2 准备maker所需要的预测基因数据库¶

    • genome：fasta格式，由上一步组装获得
    • EST：EST 原指表达序列标签(expressedsequencetags, *EST*s)，是指从不同组织来源的cDNA序列，在当下可以用组装的转录本代替。
    • altest：同源物种的EST，fasta格式
    • protein：同源物种的蛋白序列，fasta格式

    组装的转录本可以由Trinity获得。

    Trinity是Broad Institute和Hebrew University of Jerusalem开发的RNA-Seq数据 转录组组装工具，包括三个模块，

    • Inchworn(尺蠖): 将RNA-seq数据组装成单个转录本，通常是主要转录亚型的全长转录本
    • Chrysalis(蛹): 这一步将上一步得到contig进行聚类，对于每个聚类构建完整的德布罗意图(*de Bruijin*graph)。每个转录本表示的是给定基因或者一组有着共同序列的基因的全部转录组成。 之后会根据图中不相交的点对全部短读数据进行拆分
    • Butterfly(蝴蝶): 并行处理各个图(graph), 追踪每个图中的短读和配对短读的路径，最后报告可变剪切亚型的全长转录本，并且区分出旁系同源基因的转录本

    其输入结果和输出结果都十分简单。

    Trinity --seqType fq --max_memory 50G  \
         --left condA_1.fq.gz,condB_1.fq.gz,condC_1.fq.gz \
         --right condA_2.fq.gz,condB_2.fq.gz,condC_2.fq.gz \
         --CPU 6  --min_kmer_cov 2 \
         --jaccard_clip

    为加快速度，可以用--normalize_max_read_cov减少高表达的基因的reads数，用--min_kmer_cov 2参数丢弃uniquely occurring kmer, 从而降低内存消耗,也可以用fastp去除低质量的reads，用cd-hit-est来去除重复的reads。

    对于真菌转录组数据，由于基因密度较高，需添加--jaccard_clip

    最终结果为Trinity.fasta文件

    1.3 运行maker¶

    maker & maker.log &

    2 整合其它软件获得的注释结果¶

    2.1 软件配置¶

    1. RepeatModeler and RepeatMasker with all dependencies (I used NCBI BLAST) and RepBase
    2. MAKER
    3. Augustus
    4. BUSCO
    5. SNAP
    6. BEDtools

    2.2 原始数据¶

    1. 组装好的基因组文件 ：genome.fasta
    2. 组装好的转录本文件：trinity.fasta
    3. 同源的蛋白质序列：protein.fasta (from NCB or uniport or JGI )
    4. repeat library (手动注释或者同源物种)

    2.3 注释重复序列¶

    1. De Novo Repeat Identification

    对于真核生物来说，注释其重复序列对于后期基因组的注释是非常重要的。虽然有很多重复序列的数据库如Repbase，但从头对其基因组进行重复序列注释依然是十分有必要的。

    通过RepeatModeler，单核单线程注释几十兆的小型基因组只需要几个小时。

    BuildDatabase -name test -engine ncbi genome.fasta
RepeatModeler -pa 1 -engine ncbi -database test

    得到两个结果文件：

    • test-families.fa : Consensus sequence

    • test-families.stk : Seed alignments

    • Full Repeat Annotation

    通过RepeatMasker，可以进行更加深入的重复序列注释。我们需要用到上一步的test-families.fa文件

    RepeatMasker -pa 1 -e ncbi -lib test-families.fa -dir mask01/ genome.fasta

    可以获得4个文件：

    • test.fasta.tbl
    • test.fasta.cat.gz
    • test.fasta.masked
    • test.fasta.out

    针对这4个文件，我们可以结合Repbase数据库，进行第二轮重复序列注释

    RepeatMasker -pa 1 -e ncbi -species fungi -dir mask02 test.fasta.masked

    同样可以获得4个文件

    • test.fasta.out.out
    • test.fasta.masked.masked
    • test.fasta.masked.cat.gz
    • test.fasta.masked.tbl

    通过一些脚本，让这些注释文件合并，用于输入给maker

    cp test.fasta.masked.masked full_mask/test.full_mask.fa
cp test.fasta.out.out full_mask/test.full_mask.out
gunzip test.fasta.cat.gz test.fasta.masked.cat.gz 
cat *.cat > full_mask/test.full_mask.cat
cd full_mask
ProcessRepeats -species fungi test.full_mask.cat
rmOutToGFF3.pl test.full_mask.out > test.full_mask.gff3 #generate gff file
grep -v -e "Satellite" -e ")n" -e "-rich" test.full_mask.gff3 > test.full_mask.out.complex.gff3 # isolate complex repeats
cat test.full_mask.complex.gff3 | perl -ane '$id; if(!/^\#/){@F = split(/\t/, $_); chomp $F[-1];$id++; $F[-1] .= "\;ID=$id"; $_ = join("\t", @F)."\n"} print $_' > test.full_mask.complex.reformat.gff3

    最后生成的gff3文件可以直接用于maker的注释。

    3.2 第二轮注释¶

    第二轮注释可以用SNAP这个软件，它的速度非常快，可以生成基因的hmm模型，需要用到第一轮注释生成的gff文件

    mkdir snap
cd snap
mv ../test_rnd1.all.maker.noseq.gff .
maker2zff test_rnd1.all.maker.noseq.gff

    可以得到两个基因组的注释文件，分别是

    • genome.ann
    • genome.dna

    参考SNAP的官方文档，可以对其进行训练

    安装SNAP

    fathom genome.ann genome.dna -gene-stats > test.gene.stats.txt
fathom genome.ann genome.dna -validate > test.validation.txt
fathom genome.ann genome.dna -categorize 1000 #将序列分类，这里的1000推荐用基因平均长度的一半，根据基因密度适当调整
fathom uni.ann uni.dna -export 1000 -plus# 只输出unigene中正链基因

    test.gene.stats.txt用于了解基因的一些基本信息，例如正链和负链的分布，外显子的数量。

    test.validation.txt用于检查基因结构的完整性。

    最终会生成一下这些文件

    • alt.ann, alt.dna (genes with alternative splicing)
    • err.ann, err.dna (genes that have errors)
    • olp.ann, olp.dna (genes that overlap other genes)
    • wrn.ann, wrn.dna (genes with warnings)
    • uni.ann, uni.dna (single gene per sequence)

    这里只用第四类的基因，也就是每个序列上只有一个基因。

    用fathom uni.ann uni.dna -export 1000 -plus只输出unigene中正链基因，这一步同样会生成四个文件

    • export.aa 每个基因的蛋白序列
    • export.ann 正链的基因结构
    • export.dna 正链的DNA
    • export.tx 每个基因的转录本

    接着让forge负责预测参数， 由于输出会很多，所以建议创建个文件夹

    mkdir params
cd params
forge ../export.ann ../export.dna
cd ..

    最后是hmm-assembler.pl构建HMM，即基因模型文件, 

    hmm-assembler.py test params/ > test.hmm

    3.3 第三轮注释¶

    第三轮注释依赖于Augusts

    提取第二轮注释结果中基因及其前后1000bp作为训练数据。

    awk -v OFS="\t" '{ if ($3 == "mRNA") print $1, $4, $5 }' ../../round1.gff |awk -v OFS="\t" '{ if ($2 < 1000) print $1, "0", $3+1000; else print $1, $2-1000, $3+1000 }' | bedtools getfasta -fi ../../genome.fasta -bed - -fo round1.all.maker.transcripts1000.fasta

    用BUSCO可以较为方便的完成对Augusts的训练，需要注意的是，不同版本的BUSCO和Augusts需要搭配使用，且参数有很大不同。为了优化BUSCO的注释，让其结果更适合给Augusts训练，需要添加--long参数

    对于非模式物种，后面需要手动修改AUGUSTS的config文件夹，所以要保证有BUSCO和AUGUSTS安装路径的读写权限。

    BUSCO.py -i round1.all.maker.transcripts1000.fasta  -o test_rnd1_maker -l fungi_odb10/ -m genome -c 8 --long -sp aspergillus_nidulans -z --augustus_parameters='--progress=true' #-sp物种需要从BUSCO官方列表里面查,这一步用8个线程需要耗费数个小时

    完成后可以在retraining_parameters文件夹下找到对应的模型文件。这里不建议改文件名，如果需要更改文件名，同时也要改文件里面对应的名字。

    文件名通常为BUSCO_XXXX_123456,在AUGUST的config/species文件夹里面新建对应的文件夹

    mkdir $AUGUSTUS_CONFIG_PATH/species/BUSCO_XXXX_123456
cp BUSCO_XXXX_123456* $AUGUSTUS_CONFIG_PATH/species/BUSCO_XXXX_123456/

    类似的，我们整理第二轮的gff文件为第三轮注释做准备。注意Augusts的参数需要更改物种为物种文件夹名称

    # transcript alignments
awk '{ if ($2 == "est2genome") print $0 }' test_rnd1.all.maker.noseq.gff > test_rnd1.all.maker.est2genome.gff
# protein alignments
awk '{ if ($2 == "protein2genome") print $0 }' test_rnd1.all.maker.noseq.gff > test_rnd1.all.maker.protein2genome.gff
# repeat alignments
awk '{ if ($2 ~ "repeat") print $0 }' test_rnd1.all.maker.noseq.gff > test_rnd1.all.maker.repeats.gff

    通过不断调整参数和迭代，可以获得多个基因组注释结果。最后一次的结果不一定是最好的，需要进行评估。

    AED_cdf_generator.pl -b 0.025 roundN.full.gff   > test.roundN.AED.distrubtion.txt

    可以通过AED的值来评估注释的结果。

    AED值在0-1之间，数字越小，代表该基因注释的越准确。一般认为AED<0.5即为比较好的结果。
